\doxysubsection{Detailed Description}
\begin{DoxyVerb}Provides access to collections of rendering methods, which
can be local, from other templates, or from imported modules.

To access a particular rendering method referenced by a
:class:`.Namespace`, use plain attribute access:

.. sourcecode:: mako

  ${some_namespace.foo(x, y, z)}

:class:`.Namespace` also contains several built-in attributes
described here.
\end{DoxyVerb}

mako.\+runtime.\+Namespace.\+get\+\_\+namespace (\begin{DoxyParamCaption}\item[{}]{self}{, }\item[{}]{uri}{}\end{DoxyParamCaption})}

\begin{DoxyVerb}Return a :class:`.Namespace` corresponding to the given ``uri``.

If the given ``uri`` is a relative URI (i.e. it does not
contain a leading slash ``/``), the ``uri`` is adjusted to
be relative to the ``uri`` of the namespace itself. This
method is therefore mostly useful off of the built-in
``local`` namespace, described in :ref:`namespace_local`.

In
most cases, a template wouldn't need this function, and
should instead use the ``<%namespace>`` tag to load
namespaces. However, since all ``<%namespace>`` tags are
evaluated before the body of a template ever runs,
this method can be used to locate namespaces using
expressions that were generated within the body code of
the template, or to conditionally use a particular
namespace.
\end{DoxyVerb}
